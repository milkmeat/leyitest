# AVA 战场管理

通过 CLI 命令调用游戏 GM 接口管理 AVA 战场副本。支持两种操作：**创建战场** 和 **加入战场**。

根据用户意图自动判断操作类型：
- 用户说 "创建"、"新建"、"开战场" → **创建战场**
- 用户说 "加入"、"移入"、"进入"、"add"、"join" → **加入战场**

---

## 操作一：创建战场

### 参数收集

| 参数 | 必填 | 说明 |
|------|------|------|
| lvl_id | 是 | 战场 ID（整数） |
| duration | 否 | 持续时长（小时），默认 24 |

### 提取规则

1. **lvl_id**: 从用户消息中提取数字作为战场 ID。例如 "id=29999"、"29999"、"战场29999" 均提取为 2999
2. **duration**: 从用户消息中提取时长。支持以下格式：
   - "24小时" / "24hour" / "24h" → 24
   - "持续1天" / "1天" → 24
   - "2小时" / "2h" → 2
   - 如果用户未指定时长，**必须使用 AskUserQuestion 询问**，提供以下选项：
     - 24 小时（推荐）
     - 2 小时
     - 1 小时
     - 其他时长（用户自定义输入）

### 执行

```bash
python src/main.py uid_ava_create <lvl_id> duration=<hours>
```

## 操作二：加入战场

"加入战场" 是一个原子操作，内部自动串联两步：先加入名单，再移入战场。

### 参数收集

| 参数 | 必填 | 说明 |
|------|------|------|
| lvl_id | 是 | 战场 ID（整数），从用户消息中提取，或从最近创建的战场 ID 推断 |
| uid | 是 | 账号 UID（整数） |
| camp_id | 否 | 阵营 ID，默认 1（我方）。若用户说 "敌方" 则为 2 |

### 执行步骤（必须按顺序执行）

**Step 1 — 加入名单**：

```bash
python src/main.py uid_ava_add <lvl_id> <uid> <camp_id>
```

- 输出 `[OK]` → 继续执行 Step 2
- 输出 `[FAIL]` → 停止，报告错误

**Step 2 — 移入战场**：

```bash
python src/main.py uid_ava_enter <lvl_id> <uid>
```

- 输出 `[OK]` → 加入成功
- 输出 `[FAIL]` → 报告错误（名单已添加但进入失败）

### 批量加入

如果用户要求将多个账号加入战场，对每个账号依次执行 Step 1 + Step 2。
如果用户要求将全部账号加入，使用快捷命令：

```bash
python src/main.py uid_ava_join_all <lvl_id>
```

---

## 结果判断

- 命令输出包含 `[OK]` → 操作成功
- 命令输出包含 `[FAIL]` → 操作失败，展示错误信息给用户

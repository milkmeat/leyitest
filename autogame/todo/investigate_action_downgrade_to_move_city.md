## 目标
评估并优化 L0 "指令降级为移城" 的行为（日志中出现 127 次）

## 背景
对战日志分析发现，L0 执行层将大量 AI 指令降级为移城操作：
- `LVL_ATTACK_BUILDING → LVL_MOVE_CITY`：86 次
- `LVL_REINFORCE_BUILDING → LVL_MOVE_CITY`：41 次

这不是 bug——L0 的设计是"如果目标太远，先移城靠近再行动"。
但 127/437 = 29% 的操作被降级，意味着近三成的行动轮次被浪费在移城上，
AI 的攻击/驻防指令要到下一轮才能真正执行。

另外，每次移城后服务器返回 `error=-161`（共 141 次），出现在 `svr_city_wall` 推送中，
疑似城墙数据同步的附带信息，不影响移城成功，但需确认。

## 根因分析
- L1 AI 没有考虑距离因素，给远距离目标下达了直接攻击/驻防指令
- L1 prompt 中可能缺少"优先选择近距离目标"或"先移城再攻击"的策略引导
- L2 战略层分配任务时没有考虑小队与目标的距离

## 任务
- [ ] 1. 统计降级发生的 loop 分布，确认是否集中在前几轮（初始部署阶段）
- [ ] 2. 检查 L1 prompt 是否包含距离感知的引导（如 march 时间阈值）
- [ ] 3. 考虑在 L1 prompt 中增加策略：如果目标 march > 60s，先下达 MOVE_CITY 而非直接攻击
- [ ] 4. 确认 error=-161 是否为已知的无害行为，是否需要处理
- [ ] 5. 评估是否需要在 L2 层面优化任务分配（就近原则）
